# PRD V2.1 — 数据治理与企业能力

## 1. 版本目标

让平台从“可用”升级为“可治理、可复用、可管控”。

本版本同时补齐两个企业高频场景：
- 渠道归因与异常预警。
- 多项目、多角色、多数据资产的治理与复用。

## 2. 目标用户与价值

| 用户 | 版本价值 |
|------|----------|
| 数据治理团队 | 能发现脏数据、统一口径、扩展维度 |
| 管理员 | 能管理角色权限、项目边界和敏感操作 |
| 增长团队 | 能做渠道追踪、归因和异常监控 |

## 3. 功能范围

| 模块 | 子能力 | 优先级 |
|------|--------|--------|
| 归因分析 | 渠道定义、追踪链接、首次/末次/线性/时间衰减归因 | P0 |
| 智能预警 | 阈值预警、异常预警、通知渠道、抑制策略 | P0 |
| 数据建模增强 | 虚拟事件、虚拟属性、维度表 | P1 |
| 数据治理 | Schema 校验、质量监控、字典管理、控制台导入/导出 | P0 |
| 企业能力 | 多项目、资产包、RBAC、审计日志、MFA、脱敏 | P0 |

## 4. 明确不在本版本范围内

- 统一指标平台。
- 自定义 SQL 查询入口。
- OpenAPI 对外能力。
- AI 助手。

## 5. 功能详细设计

### 5.1 归因分析与渠道追踪

#### 5.1.1 场景定义

用于评估广告、活动、外部链接等触点对转化结果的贡献，并支持生成可跟踪的营销链接。

#### 5.1.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| ATTR-001 | 渠道管理 | 维护渠道、活动、素材等信息 |
| ATTR-002 | 追踪链接 | 生成带渠道参数的可追踪链接 |
| ATTR-003 | 归因模型 | 首次、末次、线性、时间衰减 |
| ATTR-004 | 归因报表 | 渠道转化数、转化金额、占比 |
| ATTR-005 | 路径查看 | 查看典型转化路径 |

#### 5.1.3 核心规则

- 所有归因分析必须显式设置归因窗口。
- 若用户在窗口内无有效触点，则计为“直接/自然流量”或“未归因”。
- 报表页面必须同时展示“转化数”和“转化金额”口径。

### 5.2 智能预警

#### 5.2.1 场景定义

用于监控关键指标是否出现异常波动，帮助团队在问题扩大前得到通知。

#### 5.2.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| ALERT-001 | 阈值预警 | 高于或低于阈值时触发 |
| ALERT-002 | 异常预警 | 与历史均值或波动区间偏离时触发 |
| ALERT-003 | 通知渠道 | 邮件、企业微信、Webhook |
| ALERT-004 | 抑制策略 | 触发后在抑制窗口内不重复提醒 |
| ALERT-005 | 历史记录 | 查看触发时间、指标值、通知状态 |

#### 5.2.3 核心规则

- 预警必须绑定明确指标或分析口径。
- 通知失败需记录原因并支持重试。
- 相同规则在抑制窗口内不得刷屏式重复通知。

### 5.3 数据建模增强

#### 5.3.1 虚拟事件

用于把现有事件做二次封装，形成更贴近业务语义的分析对象。

典型示例：
- 将 `order_paid` 中 `amount > 1000` 的记录定义为“高客单支付”。
- 将多个注册成功事件合并成统一的“注册完成”。

#### 5.3.2 虚拟属性

用于基于已有属性生成可复用的新属性，例如：
- 从 URL 中提取页面模块。
- 将订单金额映射到客单价区间。

#### 5.3.3 维度表

用于引入外部业务维度并关联到分析链路，例如：
- 商品维度表。
- 门店维度表。
- 渠道层级表。

#### 5.3.4 核心规则

- 虚拟事件和虚拟属性不应改变原始数据，只改变分析视角。
- 维度表关联必须明确主键字段和刷新频率。
- 当虚拟定义失效时，应对依赖分析给出告警提示。

### 5.4 数据治理

#### 5.4.1 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| GOV-001 | Schema 校验 | 校验事件名、属性类型、必填字段 |
| GOV-002 | 质量监控 | 查看失败事件数、告警事件数、最近问题趋势 |
| GOV-003 | 字典管理 | 管理枚举值到业务显示值的映射 |
| GOV-004 | 数据导入 | 通过控制台导入历史事件或维度数据 |
| GOV-005 | 数据导出 | 按事件、属性和时间范围导出数据 |

#### 5.4.2 核心规则

- 导入任务必须先进行字段映射预览。
- 高风险字段变更应提示对已有分析和分群的影响。
- 数据治理台需突出“影响范围”，而不是仅显示技术错误。

### 5.5 企业能力

#### 5.5.1 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| ENT-001 | 多项目管理 | 同一租户下管理多个业务项目 |
| ENT-002 | 资产包 | 书签、看板、分群、标签等跨项目复制 |
| ENT-003 | RBAC | 按角色控制模块与数据权限 |
| ENT-004 | 审计日志 | 记录登录、导出、权限变更、规则变更 |
| ENT-005 | MFA | 高权限账号支持二次验证 |
| ENT-006 | 数据脱敏 | 对手机号、邮箱等字段做掩码展示 |

#### 5.5.2 权限规则

- 查看权限、编辑权限、导出权限需要分离。
- 高危操作如资产导入、项目重置、权限变更必须写入审计日志。
- 敏感数据默认脱敏，只有被授权角色可查看明文。

## 6. 跨模块关系

- 归因分析可被看板、报表和预警复用。
- 虚拟事件、虚拟属性、维度表会进入分析、分群、标签等上游定义层。
- 资产包可复用书签、看板、分群、标签等资产。

## 7. 验收标准

### 7.1 功能验收

- 可生成追踪链接并在归因报表中看到对应渠道结果。
- 可创建阈值预警和异常预警规则，并成功发送通知。
- 可创建虚拟事件、虚拟属性和维度表并用于分析。
- 控制台支持数据导入、导出和 Schema 质量查看。
- 多项目、角色权限、审计日志、MFA 能正常工作。

### 7.2 数据正确性验收

- 归因模型切换后结果符合对应口径定义。
- 虚拟事件不改变原始事件数量，只改变分析命中范围。
- 字典映射、维度表关联后的展示值与业务预期一致。
- 脱敏规则生效时，未授权角色无法看到敏感原值。

### 7.3 企业能力验收

- 资产包导入后，依赖关系和引用对象可被识别和提示。
- 所有高风险操作均可在审计日志中追溯到操作者和时间。
