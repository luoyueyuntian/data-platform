# OIP — 优化版用户行为分析平台 PRD

## 文档定位

`oip-prd` 是一套面向“类神策分析”产品的优化版 PRD 文档集。

这套文档吸收了两类优点：
- 保留完整能力地图：覆盖数据接入、行为分析、用户洞察、运营闭环、数据治理、开放平台、AI 助手。
- 强化版本可执行性：按更清晰的阶段拆分范围，避免单个版本承载过多能力。

## 优化原则

1. 版本边界清晰：每个版本只承诺一个可交付主题。
2. 文档表达统一：统一采用“目标 -> 范围 -> 详细设计 -> 接口 -> 验收”结构。
3. 产品视角优先：重点写清业务价值、页面行为、规则口径、协作边界，避免过度陷入底层实现。
4. 兼顾完整性与节奏：能力版图尽量完整，但不把所有能力堆进早期版本。
5. 验收可检查：尽量将验收写成可验证的结果，而不是笼统描述。

## 各版本摘要

| 版本 | 目标 | 核心用户 | 产出结果 |
|------|------|----------|----------|
| V1.0 | 跑通采集、建模、事件分析、看板闭环 | 产品经理、分析师、实施人员 | 平台可开始接入真实业务数据 |
| V1.1 | 补齐最常用分析模型 | 产品经理、增长分析师 | 平台具备基础转化与留存分析能力 |
| V1.2 | 从事件走向用户 | 运营、分析师、增长团队 | 能按用户理解路径、会话和群体 |
| V2.0 | 从洞察走向运营 | 运营、CRM、增长团队 | 可做标签、分群触达与实验闭环 |
| V2.1 | 进入企业级治理阶段 | 管理员、数据治理团队、渠道团队 | 数据资产可治理、可复用、可管控 |
| V2.2 | 建立平台化和智能化能力 | 全员、外部系统集成方 | 指标统一、开放集成、AI 辅助决策 |

## 阅读建议

1. 先读 `00-PRD-总览.md`，建立全局能力地图和版本顺序。
2. 再按版本读取单文档，避免跨版本混淆能力边界。
3. 当某个能力跨版本演进时，以更晚版本的定义为准。

## 优先级说明

- `P0`：必须交付，否则该版本目标不成立。
- `P1`：建议交付，能显著提升体验或闭环。
- `P2`：可延期到下一小版本，不影响主目标上线。
