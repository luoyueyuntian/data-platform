# PRD V2.0 — 智能运营闭环

## 1. 版本目标

在已有分析和分群能力基础上，构建“标签 -> 受众 -> 触达 -> 实验 -> 复盘”的运营闭环。

本版本的核心不是新增更多分析模型，而是把洞察真正转化为可执行的运营动作。

## 2. 目标用户与价值

| 用户 | 版本价值 |
|------|----------|
| CRM/运营团队 | 能快速定义人群、执行触达并看效果 |
| 增长团队 | 能通过实验验证策略和内容效果 |
| 业务负责人 | 能看到生命周期结构变化和运营结果 |

## 3. 功能范围

| 模块 | 子能力 | 优先级 |
|------|--------|--------|
| 标签体系 | 数值标签、枚举标签、状态标签、标签分类 | P0 |
| 生命周期 | 阶段定义、阶段人数、阶段流转、趋势 | P0 |
| 受众服务 | 实时受众、非实时受众、受众预估、在线判定 | P0 |
| 触达计划 | 邮件、短信、App Push、Webhook、定时发送 | P0 |
| 旅程编排 | 触发节点、等待节点、条件分支、结束节点 | P1 |
| A/B 测试 | 分流、目标指标、实验报告、显著性判断 | P0 |
| 热力图 | Web 点击热力图、页面筛选、快照管理 | P1 |

## 4. 明确不在本版本范围内

- 多项目资产打包。
- 指标平台、自定义 SQL、OpenAPI。
- AI 自动圈人和 AI 文案生成。

## 5. 功能详细设计

### 5.1 标签体系

#### 5.1.1 场景定义

标签是对用户特征的结构化沉淀，既可用于画像展示，也可用于受众筛选和触达决策。

#### 5.1.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| TAG-001 | 标签定义 | 支持名称、类型、描述、归属分类 |
| TAG-002 | 数值标签 | 例如近 30 天支付金额、近 7 天访问次数 |
| TAG-003 | 枚举标签 | 例如会员等级、渠道类型、偏好品类 |
| TAG-004 | 状态标签 | 例如活跃、沉默、流失风险 |
| TAG-005 | 刷新策略 | 实时、每小时、每天 |

#### 5.1.3 核心规则

- 标签必须有明确刷新频率。
- 标签值变更应可追踪最近更新时间。
- 同一用户同一标签在任一时刻只允许存在一个有效值。

### 5.2 生命周期

#### 5.2.1 场景定义

生命周期用于把用户按业务活跃阶段划分，并观察阶段间迁移情况。

#### 5.2.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| LIFE-001 | 阶段配置 | 新用户、活跃、沉默、流失、回流等可配置 |
| LIFE-002 | 阶段人数 | 查看各阶段当前人数与变化趋势 |
| LIFE-003 | 阶段流转 | 展示阶段迁移关系 |
| LIFE-004 | 关键指标 | 新增、活跃、沉默、流失、回流 |

#### 5.2.3 核心规则

- 生命周期规则必须显式依赖事件和时间窗口。
- 阶段之间默认互斥，同一用户同一时间只能处于一个主阶段。

### 5.3 受众服务

#### 5.3.1 场景定义

受众是可直接用于运营动作执行的人群容器，可由一个或多个分群、标签条件组合而成。

#### 5.3.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| AUD-001 | 受众创建 | 支持从分群、标签、生命周期阶段组合创建 |
| AUD-002 | 实时受众 | 用于在线判定和实时触达 |
| AUD-003 | 非实时受众 | 用于批量营销和离线任务 |
| AUD-004 | 人数预估 | 发送前估算受众规模 |
| AUD-005 | 在线判定 | 提供接口判断某用户是否属于某受众 |

### 5.4 触达计划

#### 5.4.1 场景定义

触达计划用于把受众转换成具体消息动作，并追踪送达、打开、点击和转化效果。

#### 5.4.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| CMP-001 | 计划创建 | 选择受众、渠道、内容、时间 |
| CMP-002 | 渠道支持 | 邮件、短信、App Push、Webhook |
| CMP-003 | 定时发送 | 立即发送、定时发送、周期发送 |
| CMP-004 | 效果统计 | 送达、失败、打开、点击、转化 |
| CMP-005 | 频控 | 限制用户在时间窗口内收到的触达次数 |

#### 5.4.3 核心规则

- 发送前必须进行人数预估和内容预览。
- 同一用户在频控窗口内超过限制时不可重复发送。
- 触达效果统计需区分“发送指标”和“转化指标”。

### 5.5 旅程编排

#### 5.5.1 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| JNY-001 | 触发节点 | 用户进入受众、完成事件、达到标签条件 |
| JNY-002 | 等待节点 | 固定时长等待或指定时间等待 |
| JNY-003 | 条件分支 | 根据标签、属性、行为分流 |
| JNY-004 | 触达节点 | 执行邮件、短信、Push、Webhook |
| JNY-005 | 结束节点 | 正常结束或超时结束 |

### 5.6 A/B 测试

#### 5.6.1 场景定义

A/B 测试用于验证产品策略、内容策略或触达策略的效果差异。

#### 5.6.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| EXP-001 | 实验创建 | 设置实验对象、变体、流量比例 |
| EXP-002 | 指标配置 | 主指标、次指标、观察周期 |
| EXP-003 | 分流算法 | 一致性分流，保证用户落桶稳定 |
| EXP-004 | 实验报告 | 展示指标差异、样本量、显著性 |
| EXP-005 | 实验互斥 | 避免同一用户同时进入冲突实验 |

#### 5.6.3 核心规则

- 同一实验内同一用户必须稳定落在同一变体。
- 默认要求每组达到最小样本量后再展示显著性提示。
- 指标必须来源于已定义分析口径，不允许临时报错口径。

### 5.7 热力图

#### 5.7.1 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| HEAT-001 | 页面快照管理 | 保存页面快照并绑定页面路径 |
| HEAT-002 | 点击热力渲染 | 用颜色深浅展示点击密度 |
| HEAT-003 | 条件筛选 | 支持按时间、设备、分群、受众筛选 |
| HEAT-004 | 元素明细 | 点击某个区域可查看点击次数和占比 |

## 6. 跨模块关系

- 用户分群是标签、受众、触达的上游输入。
- 标签和生命周期都可作为受众筛选条件。
- 受众可作为实验对象或触达对象。
- 触达结果可回流到事件分析、漏斗、归因和生命周期中。

## 7. 验收标准

### 7.1 功能验收

- 标签支持定义、刷新和在用户画像中展示。
- 生命周期可配置并输出阶段人数与流转结果。
- 受众支持实时与非实时两类配置。
- 触达计划可按渠道发送并输出基本效果统计。
- A/B 测试支持多变体实验和稳定分流。
- 热力图支持页面级点击聚合和筛选。

### 7.2 数据正确性验收

- 标签刷新后，用户标签值与定义规则一致。
- 生命周期阶段互斥且迁移可解释。
- A/B 分流对同一用户具备稳定性。
- 触达转化结果与后续事件数据能正确关联。

### 7.3 运营可用性验收

- 运营人员可不依赖研发完成一次“建受众 -> 发触达 -> 看效果”的闭环。
- 实验报告中主指标、样本量、显著性提示清晰可读。
