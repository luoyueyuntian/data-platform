# PRD V1.0 — 数据接入与基础分析

## 1. 版本目标

完成数据接入、元数据管理、事件分析、基础看板、项目与成员基础能力，跑通平台最小可用闭环。

版本成功标准：
- 客户可完成首个项目接入。
- 平台可稳定接收并校验事件数据。
- 用户可在界面完成事件分析并保存到看板。

## 2. 目标用户与价值

| 用户 | 版本价值 |
|------|----------|
| 实施人员 | 可快速完成项目创建、Token 配置、SDK 调试 |
| 产品经理 | 可查看核心事件趋势和分组结果 |
| 数据分析师 | 可做基础筛选、分组、对比和看板搭建 |
| 管理员 | 可管理项目、成员、角色和接入凭证 |

## 3. 功能范围

| 模块 | 子能力 | 优先级 |
|------|--------|--------|
| 数据接入 | Web SDK、服务端 SDK、HTTP 上报、批量上报、调试校验 | P0 |
| 元数据管理 | 元事件、事件属性、用户属性、属性字典说明 | P0 |
| 事件分析 | 趋势分析、指标选择、属性筛选、分组、对比 | P0 |
| 基础看板 | 看板创建、图表添加、布局调整、自动刷新 | P0 |
| 项目基础能力 | 项目创建、项目 Token、成员角色、基础审计记录 | P0 |
| 分析效率 | 常用时间范围、查询历史 | P1 |

## 4. 明确不在本版本范围内

- 漏斗、留存、路径、会话、分群。
- 标签、触达、实验。
- 多项目治理、指标平台、OpenAPI、AI。

## 5. 功能详细设计

### 5.1 数据接入

#### 5.1.1 目标

让客户能以最小成本完成首批事件接入，并在接入初期快速发现事件命名、属性类型、身份标识问题。

#### 5.1.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| ING-001 | Web SDK | 支持页面浏览、元素点击、自定义事件、用户属性设置 |
| ING-002 | 服务端 SDK | 支持服务端埋点、用户属性写入、批量发送 |
| ING-003 | HTTP 上报 | 提供事件上报接口，支持单条和批量模式 |
| ING-004 | 调试模式 | 返回字段级校验结果，不进入正式分析链路 |
| ING-005 | 失败重试 | SDK 支持重试与本地缓存，避免临时网络丢数 |
| ING-006 | Token 校验 | 每个项目独立 Token，用于接入鉴权 |

#### 5.1.3 关键页面

1. 接入概览页
- 展示当前项目 Token、上报地址、最近 24 小时接入量。
- 提供 SDK 文档入口与示例代码。
- 展示最近调试失败事件。

2. 事件调试页
- 可实时查看调试事件。
- 按校验状态筛选：成功、告警、失败。
- 点击单条事件可展开字段明细和错误提示。

#### 5.1.4 业务规则

- 事件名必须唯一且可读，禁止仅靠数字区分。
- 事件属性首次入库后锁定属性类型，后续类型冲突记为异常。
- 同一事件允许有可选属性，不强制每条都完整。
- 调试模式事件不计入正式分析结果。
- 匿名用户与登录用户至少要存在一个标识。

#### 5.1.5 核心接口

**事件上报**：`POST /api/v1/track`

```json
{
  "project_id": "proj_001",
  "events": [
    {
      "event_name": "$pageview",
      "time": 1715232000000,
      "distinct_id": "anon_001",
      "user_id": "user_001",
      "properties": {
        "$url": "/home",
        "$title": "首页"
      }
    }
  ]
}
```

**调试校验**：`POST /api/v1/track/debug`

```json
{
  "project_id": "proj_001",
  "events": [
    {
      "event_name": "order_paid",
      "distinct_id": "anon_001",
      "properties": {
        "amount": "99"
      }
    }
  ]
}
```

### 5.2 元数据管理

#### 5.2.1 目标

建立事件和属性的统一语义，使分析结果可被不同角色理解和复用。

#### 5.2.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| META-001 | 元事件管理 | 查看、创建、编辑、停用元事件定义 |
| META-002 | 事件属性管理 | 维护属性名、显示名、数据类型、说明、示例值 |
| META-003 | 用户属性管理 | 维护用户属性的展示名、说明和可用状态 |
| META-004 | 属性搜索 | 可按事件、属性名、类型进行筛选 |
| META-005 | 事件说明协作 | 支持事件描述、负责人、业务标签 |

#### 5.2.3 页面规则

- 事件名不可随意修改，避免历史分析失真。
- 可修改显示名和说明，不影响原始事件。
- 停用事件后，历史数据仍可查询，但新建分析默认不推荐。

### 5.3 事件分析

#### 5.3.1 目标

提供平台最基础也最高频的分析入口，用于查看事件趋势、用户数、分组差异和时间对比。

#### 5.3.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| EA-001 | 指标选择 | 总次数、用户数、人均次数、属性求和、均值 |
| EA-002 | 事件筛选 | 支持按事件属性和用户属性筛选 |
| EA-003 | 分组 | 支持按一个或两个属性分组 |
| EA-004 | 时间对比 | 支持上一周期、自定义时间段对比 |
| EA-005 | 图表切换 | 折线图、柱状图、表格、数值卡片 |
| EA-006 | 明细查看 | 可查看每个时间点或分组的明细值 |

#### 5.3.3 指标口径

| 指标 | 定义 |
|------|------|
| 总次数 | 满足条件的事件总量 |
| 用户数 | 触发该事件的去重用户数 |
| 人均次数 | 总次数 / 用户数 |
| 属性求和 | 对数值型属性做求和 |
| 属性均值 | 对数值型属性做平均 |

#### 5.3.4 页面布局

```text
[事件选择] [指标选择] [时间范围] [对比]
[属性筛选] [用户属性筛选] [分组] [查询]
[图表区]
[明细区] [保存到看板]
```

#### 5.3.5 核心接口

**事件分析查询**：`POST /api/v1/analysis/events`

```json
{
  "project_id": "proj_001",
  "event_name": "order_paid",
  "metric": "unique_users",
  "filters": [
    { "property": "amount", "operator": ">=", "value": 100 }
  ],
  "group_by": ["channel"],
  "compare": "previous_period",
  "date_range": {
    "start": "2025-04-01",
    "end": "2025-04-30"
  },
  "granularity": "day"
}
```

### 5.4 基础看板

#### 5.4.1 目标

把单次分析结果转成稳定复用的业务看板，支持日常查看和团队协作。

#### 5.4.2 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| DASH-001 | 看板创建 | 新建、编辑、删除看板 |
| DASH-002 | 图表添加 | 将事件分析结果添加到看板 |
| DASH-003 | 布局管理 | 组件拖拽、尺寸调整、排序 |
| DASH-004 | 自动刷新 | 支持关闭、5 分钟、15 分钟、1 小时 |
| DASH-005 | 只读访问 | 查看者可只读访问看板 |

#### 5.4.3 页面规则

- 看板内组件默认沿用原分析条件。
- 组件刷新失败时展示错误状态，不影响其他组件加载。
- 看板首页支持设为“项目默认首页”。

### 5.5 项目基础能力

#### 5.5.1 核心能力

| 需求 ID | 功能 | 描述 |
|---------|------|------|
| PROJ-001 | 项目创建 | 创建分析项目，生成唯一项目标识 |
| PROJ-002 | Token 管理 | 生成和重置项目接入 Token |
| PROJ-003 | 成员管理 | 邀请成员并配置基础角色 |
| PROJ-004 | 登录鉴权 | 支持基础注册、登录、登出 |
| PROJ-005 | 审计记录 | 记录项目创建、Token 重置、成员变更 |

#### 5.5.2 基础角色权限

| 角色 | 权限 |
|------|------|
| owner | 项目完全控制 |
| admin | 管理成员、配置接入、查看全部分析 |
| analyst | 创建分析和看板 |
| viewer | 查看分析和看板 |

## 6. 关键依赖关系

- 看板依赖事件分析查询结果。
- 元事件和属性定义依赖接入数据进入平台。
- 项目 Token 是 SDK 和上报接口接入前提。

## 7. 验收标准

### 7.1 功能验收

- 可创建项目并生成独立接入 Token。
- Web SDK 可采集 `$pageview`、`$click` 和自定义事件。
- 调试模式可对错误字段返回明确报错原因。
- 元事件、事件属性、用户属性可正常创建和停用。
- 事件分析可按属性筛选、分组和时间对比输出结果。
- 事件分析结果可添加到看板并正确保存布局。
- 不同角色看到的操作按钮符合权限预期。

### 7.2 数据正确性验收

- 用户数采用去重用户口径，不得与总次数混淆。
- 属性型指标对非数值属性应明确报错，不可静默计算。
- 调试事件不得出现在正式分析结果中。

### 7.3 性能验收

- 接入调试页能在 10 秒内看到新到达的调试事件。
- 单图表看板打开时间 P95 小于 5 秒。
- 近 30 天事件趋势查询 P95 小于 3 秒。
