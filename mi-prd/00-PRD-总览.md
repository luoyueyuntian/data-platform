# PRD 总览：用户行为分析平台（类神策分析）

## 1. 产品愿景

构建一个 SaaS 化的用户行为数据分析平台，对标神策分析（Sensors Analytics），提供从数据采集、用户标识、行为分析到智能运营的全链路能力。

## 5. 通用约定

### 5.1 数据模型基础

所有分析基于**事件模型（Event Model）**：

```
Event {
  event_id: string          // 事件唯一 ID
  event_name: string        // 事件名称（如 $pageview, $click, order_created）
  user_id: string           // 用户标识
  distinct_id: string       // 设备标识（匿名用户）
  properties: JSON          // 事件属性（动态 KV）
  timestamp: DateTime       // 事件发生时间
  project_id: string        // 项目/租户 ID
}
```

### 5.2 用户标识规则

- **登录用户**：使用 `user_id`（业务系统分配）
- **匿名用户**：使用 `distinct_id`（SDK 自动生成）
- **ID 映射**：支持 `alias` 和 `bind` 操作，将匿名 ID 关联到登录用户

### 5.3 API 响应格式

```json
{
  "code": 0,
  "message": "ok",
  "data": { ... }
}
```

### 5.4 权限模型

- **租户（Tenant）**：数据隔离边界
- **项目（Project）**：一个租户可有多个项目
- **角色（Role）**：管理员 / 分析师 / 查看者
- **权限粒度**：功能模块 + 数据范围

### 5.5 前端技术栈

- Next.js 15 + React 19
- TailwindCSS + shadcn/ui
- ECharts（图表）
- TanStack Query（数据请求）
- Zustand（状态管理）
