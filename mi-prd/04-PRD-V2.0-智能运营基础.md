## 3. 功能详细设计

### 3.1 用户分群

#### 3.1.1 概念说明

用户分群是根据用户属性和行为条件，将用户划分为不同群体的能力。例如：
- 高价值用户：近 30 天消费 > 1000 元
- 沉默用户：近 30 天无登录
- 新注册用户：注册时间在近 7 天

#### 3.1.2 分群类型

**条件分群**：
- 基于用户属性条件（如 vip_level >= 3）
- 基于行为条件（如 order_created 事件次数 >= 5）
- 支持 AND/OR 逻辑组合
- 支持时间窗口限定

**状态分群**：
- **实时分群**：条件满足时立即加入，不满足时立即移除
- **定时分群**：按计划定期更新（每小时/每天/每周）

#### 3.1.4 条件类型

**用户属性条件**：
| 操作符 | 适用类型 | 说明 |
|--------|----------|------|
| = | 所有 | 等于 |
| != | 所有 | 不等于 |
| >, <, >=, <= | 数字 | 比较 |
| 包含 | 字符串 | 包含子串 |
| 不包含 | 字符串 | 不包含子串 |
| 有值 | 所有 | 属性存在且非空 |
| 无值 | 所有 | 属性不存在或为空 |
| 在...中 | 所有 | 值在列表中 |
| 不在...中 | 所有 | 值不在列表中 |

**行为条件**：
| 指标 | 说明 |
|------|------|
| 总次数 | 事件触发总次数 |
| 用户数 | 触发事件的独立用户数（通常为 1） |
| 天数 | 触发事件的天数 |
| 首次时间 | 首次触发时间 |
| 最近时间 | 最近触发时间 |
| 属性值 | 事件属性的聚合值（sum/avg/max/min） |

#### 3.1.5 API 设计

**创建分群**：`POST /api/v1/segments`

```json
{
  "project_id": "proj_001",

  "description": "近30天消费超过1000元的VIP用户",
  "update_frequency": "realtime",
  "conditions": {
    "logic": "and",
    "rules": [
      {
        "type": "user_property",
        "property": "vip_level",
        "operator": ">=",
        "value": 3

      {
        "type": "event",
        "event": "order_paid",

        "operator": ">=",

          "value": 30,
          "unit": "day"

    ]
  }
}
```

### 3.2 用户生命周期

#### 3.2.1 概念说明

用户生命周期将用户划分为不同阶段，帮助企业理解用户状态并制定运营策略。

**标准生命周期阶段**：
| 阶段 | 定义 | 颜色 |
|------|------|------|
| 新增 | 首次访问在近 7 天 | 绿色 |
| 活跃 | 近 7 天有访问 | 蓝色 |
| 沉默 | 近 7-30 天无访问 | 黄色 |
| 流失 | 近 30 天以上无访问 | 红色 |
| 回流 | 曾经流失，近期重新访问 | 紫色 |

### 3.3 A/B 测试

#### 3.3.1 概念说明

A/B 测试用于验证产品改动的效果。通过将用户随机分配到不同实验组，对比各组在关键指标上的差异。

**核心概念**：
- **实验（Experiment）**：一次 A/B 测试
- **变体（Variant）**：实验的不同版本（对照组/实验组）
- **分流（Traffic Split）**：用户分配到各变体的比例
- **指标（Metric）**：用于评估实验效果的指标

#### 3.3.3 业务规则

**分流算法**：
- 使用一致性哈希：`hash(user_id + experiment_id) % 100`
- 确保同一用户在同一实验中始终分到同一变体
- 流量分配支持任意比例（如 10%/30%/60%）

**统计显著性**：
- 使用 Z 检验（比例型指标）或 T 检验（均值型指标）
- 默认置信度阈值：95%
- 最小样本量：每组 1000 用户

**实验生命周期**：
- **草稿**：配置中，未启动
- **运行中**：正在收集数据
- **暂停**：临时停止收集数据
- **已结束**：实验完成，得出结论

#### 3.3.5 实现要点

**分流算法**：
```typescript
function assignVariant(userId: string, experimentId: string, variants: Variant[]): Variant {
  const hash = murmurhash3(`${userId}:${experimentId}`);
  const bucket = hash % 100;
  let cumulative = 0;
  for (const variant of variants) {
    cumulative += variant.traffic_percentage;
    if (bucket < cumulative) {
      return variant;
    }
  }
  return variants[variants.length - 1];
}
```

**统计显著性计算**：
```typescript
function calculateSignificance(control: MetricResult, treatment: MetricResult): AnalysisResult {
  const p1 = control.value;
  const p2 = treatment.value;
  const n1 = control.users;
  const n2 = treatment.users;

  const pPool = (p1 * n1 + p2 * n2) / (n1 + n2);
  const se = Math.sqrt(pPool * (1 - pPool) * (1 / n1 + 1 / n2));
  const z = (p2 - p1) / se;
  const pValue = 2 * (1 - normalCDF(Math.abs(z)));

  return {
    lift: (p2 - p1) / p1,
    p_value: pValue,
    confidence: 1 - pValue,
    is_significant: pValue < 0.05
  };
}
```
