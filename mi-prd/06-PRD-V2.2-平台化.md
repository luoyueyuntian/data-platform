#### 3.1.2 事件校验

**Schema 定义**：
```json
{
  "event_name": "order_paid",
  "properties": [
    {
      "name": "order_id",
      "type": "string",
      "required": true,
      "description": "订单ID"
    },
    {
      "name": "amount",
      "type": "number",
      "required": true,
      "min": 0,
      "max": 999999,
      "description": "订单金额"
    },
    {
      "name": "items",
      "type": "array",
      "required": false,
      "description": "订单商品列表"
    }
  ]
}
```

**校验规则**：
- 必填属性检查
- 类型检查（string/number/boolean/array）
- 范围检查（min/max）
- 格式检查（正则表达式）
- 枚举值检查

**校验模式**：
- **严格模式**：不合规数据直接丢弃
- **宽松模式**：不合规数据仍入库，但标记为异常

#### 3.1.3 数据导入/导出

**数据导入**：
```
POST /api/v1/data/import
Content-Type: multipart/form-data

file: events.csv
format: csv/json
mapping: {
  "user_id": "uid",
  "event_name": "event",
  "timestamp": "ts"
}
```

**CSV 格式要求**：
```csv
event_name,user_id,timestamp,properties
pageview,u001,2025-04-30T10:00:00Z,"{""url"":""/home""}"
order_paid,u001,2025-04-30T10:05:00Z,"{""amount"":99}"
```

**数据导出**：
```
POST /api/v1/data/export
{
  "project_id": "proj_001",
  "event_name": "order_paid",
  "date_range": {
    "start": "2025-04-01",
    "end": "2025-04-30"
  },
  "format": "csv",
  "fields": ["user_id", "timestamp", "amount", "category"]
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "export_id": "export_001",
    "status": "processing",
    "estimated_rows": 120000,
    "download_url": null
  }
}
```

#### 3.1.4 字典管理

**用途**：将内部属性值映射为可读标签

**示例**：
| 属性 | 原始值 | 显示值 |
|------|--------|--------|
| channel | organic | 自然流量 |
| channel | paid_search | 付费搜索 |
| channel | social | 社交媒体 |
| vip_level | 1 | 普通用户 |
| vip_level | 2 | 银卡用户 |
| vip_level | 3 | 金卡用户 |

**API**：
```
GET    /api/v1/dictionaries                    # 字典列表
POST   /api/v1/dictionaries                    # 创建字典
PUT    /api/v1/dictionaries/:id                # 更新字典
DELETE /api/v1/dictionaries/:id                # 删除字典
```

#### 3.2.2 分析查询 API

**通用查询接口**：`POST /api/v2/query`

```json
{
  "query_type": "event_analysis",
  "params": {
    "event_name": "order_created",
    "metrics": ["total", "unique_users"],
    "date_range": {
      "start": "2025-04-01",
      "end": "2025-04-30",
      "granularity": "day"
    },
    "filters": [
      { "property": "amount", "operator": ">=", "value": 100 }
    ],
    "group_by": "category"
  }
}
```

**响应**：
```json
{
  "code": 0,
  "data": {
    "columns": ["date", "category", "total", "unique_users"],
    "rows": [
      ["2025-04-01", "手机", 156, 89],
      ["2025-04-01", "电脑", 98, 67],
      ["2025-04-02", "手机", 178, 102]
    ],
    "total_rows": 60,
    "query_time_ms": 1234
  }
}
```

**支持的查询类型**：
| 类型 | 说明 |
|------|------|
| event_analysis | 事件分析 |
| funnel | 漏斗分析 |
| retention | 留存分析 |
| distribution | 分布分析 |
| path | 路径分析 |
| session | 会话分析 |
| attribution | 归因分析 |
| user_list | 用户列表 |
| user_detail | 用户详情 |
| segment_users | 分群用户 |

#### 3.2.3 Webhook 回调

**用途**：当特定条件满足时，向外部系统发送 HTTP 请求

**配置**：
```
POST /api/v1/webhooks
{
  "name": "订单通知",
  "url": "https://external-system.com/webhook",
  "events": ["alert.triggered", "segment.updated"],
  "secret": "webhook_secret_key",
  "headers": {
    "X-Custom-Header": "value"
  }
}
```

**Webhook 负载**：
```json
{
  "event": "alert.triggered",
  "timestamp": "2025-04-30T14:30:00Z",
  "data": {
    "alert_id": "alert_001",
    "alert_name": "订单量下降预警",
    "current_value": 850,
    "threshold": 1000
  }
}
```

**签名验证**：
```
X-Webhook-Signature: sha256=<HMAC-SHA256(secret, body)>
```
