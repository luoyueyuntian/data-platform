# V0.2 PRD - 核心分析 MVP

## 1. 版本目标

V0.2 的目标是让业务团队在数据已接入的基础上，真正开始自助分析。上线后，产品、运营、分析师应能完成事件分析、属性分析、漏斗分析、留存分析，并将关键结果保存到概览中持续查看。

## 2. 用户价值

1. 产品经理可评估功能使用趋势与转化效果
2. 运营可查看用户构成和核心行为留存
3. 管理层可通过概览快速看关键指标
4. 分析师可复用分析配置并统一表达分析口径

## 3. In Scope

1. 事件分析
2. 属性分析
3. 漏斗分析
4. 留存分析
5. 基础工作台
6. 概览 / Dashboard
7. 分析书签与查询保存
8. 基础图表导出
9. 结果下钻到用户列表的预留能力

## 4. Out of Scope

1. 分群正式管理
2. 路径分析
3. 分布分析
4. 归因分析
5. 智能预警
6. 标签系统

## 7. 查询模型要求

### 7.1 通用查询结构

所有分析模型都应遵循统一的 Query DSL：

1. `subject` 分析主体
2. `metrics` 指标
3. `filters` 全局条件
4. `breakdowns` 分组维度
5. `timeRange` 时间范围
6. `granularity` 时间粒度
7. `compare` 对比维度或对比时间

### 7.2 查询执行要求

1. 长查询必须支持异步执行
2. 前端需支持轮询任务状态
3. 系统可对高频相同查询做缓存
4. 每次查询需保留执行耗时和执行状态

## 8. 页面交互要求

1. 查询条件区与结果区分离
2. 支持“收起默认项”模式
3. 对高耗时模型提供主动查询按钮
4. 图表与表格可切换
5. 鼠标悬停时展示指标说明和口径提示
