# OI Analytics Cloud PRD

本文档集用于定义一个面向 SaaS 场景、能力边界参考神策分析（Sensors Analytics）产品手册的产品需求说明书（PRD）体系。

目标不是机械复刻某一产品，而是提炼其成熟能力结构，输出一套适合 AI 直接进入设计、开发、测试与拆解的产品文档。

## 使用方式

建议 AI 开发时按以下顺序读取：

1. `00-产品愿景与边界.md`
2. `01-版本路线图.md`

## 产品工作名

`OI Analytics Cloud`

## 目标用户

1. 中小到中大型互联网产品团队
2. 需要行为分析、用户分群、运营评估、渠道归因的增长团队
3. 有多项目、多成员、多角色协作需求的数据团队
4. 希望先用标准 SaaS 版，再逐步扩展为数据中台能力的企业

## 能力分层

1. `V0.1` 数据接入与元数据治理
2. `V0.2` 核心行为分析 MVP
3. `V1.0` 增长分析与团队协作
4. `V2.0` 用户资产沉淀与智能化分析

## 交付原则

1. 每个版本都要求可独立上线，而不是只完成半成品底座
2. 每个版本都包含明确的页面、接口、数据对象、权限、验收标准
3. 每个版本都默认支持 SaaS 多租户模式
4. 每个版本都可被 AI 进一步拆解为 Epic、Story、Task、API、测试用例

## 不在本轮 PRD 中展开的范围

1. A/B Testing 独立产品线
2. 推荐系统独立产品线
3. 营销自动化编排器的完整营销云能力
4. BI 自助 SQL IDE
5. 可视化埋点 SDK 设计细节到各端原生实现层
