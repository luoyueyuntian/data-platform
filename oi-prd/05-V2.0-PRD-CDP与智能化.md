# V2.0 PRD - CDP 与智能化

## 1. 版本目标

V2.0 的目标是把平台从分析系统升级成用户资产和统一口径平台。上线后，企业应能沉淀标签、画像、归因、指标口径和智能诊断结果，让分析与运营真正形成长期资产。

## 2. 用户价值

1. 数据团队可建立统一标签体系和统一指标口径
2. 运营团队可直接使用标签和画像做精细化运营
3. 市场团队可用归因分析优化投放预算
4. 管理层可通过统一指标与异常诊断快速识别问题

## 3. In Scope

1. 标签系统
2. 用户画像视图
3. 用户群画像分析
4. 归因分析
5. 指标平台基础版
6. 指标分析
7. 智能预警增强版
8. 异常原因诊断
9. 开放 API 与 Webhook 扩展

## 4. Out of Scope

1. 完整营销自动化编排器
2. 推荐算法平台
3. 复杂数据开发工作台
4. 自助 SQL 研发平台

## 5. 页面清单

| 页面 | 优先级 | 说明 |
| --- | --- | --- |
| 标签管理 | P0 | 创建、管理、审批、版本记录 |
| 用户画像详情 | P0 | 用户属性、标签、行为摘要 |
| 用户群画像分析 | P1 | 对比不同人群画像差异 |
| 归因分析 | P0 | 渠道 / 触点贡献度分析 |
| 指标平台 | P0 | 指标、维度、关系配置 |
| 指标分析 | P0 | 统一口径即席分析 |
| 异常诊断中心 | P1 | 异常维度与异常用户诊断 |
| 开放平台 | P1 | API Key、Webhook、事件订阅 |

## 6. 功能需求

### 6.1 标签系统

| 编号 | 需求 |
| --- | --- |
| R-V20-001 | 支持规则创建标签 |
| R-V20-002 | 支持枚举型、数值分层型、布尔型标签 |
| R-V20-003 | 支持手动计算与例行计算 |
| R-V20-004 | 支持标签目录分组 |
| R-V20-005 | 支持标签版本与规则变更记录 |
| R-V20-006 | 标签可在分群、分析、画像中复用 |

### 6.2 用户画像

| 编号 | 需求 |
| --- | --- |
| R-V20-010 | 用户画像页需展示基础属性、标签、最近行为、关键事件 |
| R-V20-011 | 支持从用户细查和分群结果跳转到画像 |
| R-V20-012 | 支持展示生命周期状态与最近活跃时间 |

### 6.3 用户群画像分析

| 编号 | 需求 |
| --- | --- |
| R-V20-020 | 支持选择一个目标用户群和多个对比用户群 |
| R-V20-021 | 支持比较属性分布、标签分布和行为分布 |
| R-V20-022 | 支持输出差异显著的维度 |

### 6.4 归因分析

| 编号 | 需求 |
| --- | --- |
| R-V20-030 | 支持定义归因事件与目标转化事件 |
| R-V20-031 | 支持至少三种归因模型：首次触点、末次触点、线性归因 |
| R-V20-032 | 支持归因窗口设置 |
| R-V20-033 | 支持按渠道、广告位、活动维度输出贡献度 |

### 6.5 指标平台

| 编号 | 需求 |
| --- | --- |
| R-V20-040 | 支持创建指标、维度、指标分组 |
| R-V20-041 | 指标需绑定口径定义与计算 SQL / DSL |
| R-V20-042 | 支持指标血缘与引用关系 |
| R-V20-043 | 指标可在指标分析和概览中复用 |

### 6.6 指标分析

| 编号 | 需求 |
| --- | --- |
| R-V20-050 | 支持通过选择指标、维度、筛选条件进行分析 |
| R-V20-051 | 支持多指标同屏对比 |
| R-V20-052 | 支持按统一口径输出图表和表格 |

### 6.7 智能预警增强版

| 编号 | 需求 |
| --- | --- |
| R-V20-060 | 除阈值规则外，支持基于历史趋势的异常区间检测 |
| R-V20-061 | 支持按小时和按天学习周期规律 |
| R-V20-062 | 支持预警详情中的异常原因分析 |

### 6.8 异常诊断

| 编号 | 需求 |
| --- | --- |
| R-V20-070 | 预警触发后，系统可自动尝试发现异常维度 |
| R-V20-071 | 可输出贡献度最高的异常维度组合 |
| R-V20-072 | 支持识别异常用户列表并下钻查看 |

### 6.9 开放平台

| 编号 | 需求 |
| --- | --- |
| R-V20-080 | 提供标准 Open API 访问标签、分群、概览、指标 |
| R-V20-081 | 支持 Webhook 订阅预警和分群结果 |
| R-V20-082 | 提供 API Key 与权限控制 |

| attribution_model | name, model_type, window_config |
| metric_definition | name, code, formula, grain, owner_id |
| diagnosis_report | source_alert_id, findings_json, status |

## 8. 验收标准

| 编号 | 验收项 |
| --- | --- |
| AC-V20-001 | 标签创建后可在分群规则中直接复用 |
| AC-V20-002 | 用户画像页可展示至少 30 天行为摘要 |
| AC-V20-003 | 归因分析可输出不同渠道对转化的贡献占比 |
| AC-V20-004 | 指标平台中定义的指标可在指标分析直接选择 |
| AC-V20-005 | 智能预警触发后可展示异常维度诊断结果 |

## 9. 版本完成定义

V2.0 完成后，OI Analytics Cloud 将具备从“分析工具”向“企业级增长与用户资产平台”升级的关键能力。
