## 7. 映射原则

1. 本 PRD 参考的是“能力结构”和“成熟交互抽象”，不是逐页复刻
2. 所有功能已按 SaaS 化、可迭代交付、适合 AI 开发拆解的方式重新组织
3. 某些神策文档中的高级收费功能，被本 PRD 重新安排到更后续版本，以保证前期实现复杂度可控

## 8. 与原能力的差异化取舍

| 取舍项 | 说明 |
| --- | --- |
| 先做规则版，后做智能版 | 例如预警、诊断、标签计算先规则化，再智能化 |
| 先做核心分析，后做统一指标 | 避免在 MVP 阶段过早建设重口径平台 |
| 先做 SaaS 通用能力 | 不提前引入过重的私有化部署特性 |
| 先做分析沉淀，后做营销编排 | 保证平台先形成稳定的数据资产 |
