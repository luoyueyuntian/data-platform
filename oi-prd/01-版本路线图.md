# 版本路线图

## 1. 规划原则

1. 先解决数据接入可信，再做分析可用，最后做运营可沉淀
2. 每个版本只交付一个清晰的业务价值闭环
3. 高复杂度能力按“规则版优先，智能版后置”的方式迭代
4. 先做高频分析模型，再做高阶建模与智能化

## 2. 版本总览

| 版本 | 核心主题 | 面向角色 | 上线目标 |
| --- | --- | --- | --- |
| V0.1 | 数据接入与治理底座 | 管理员、研发 | 可以稳定接入和管理事件数据 |
| V0.2 | 核心分析 MVP | 产品、运营、分析师 | 可以完成基础自助分析与看板沉淀 |
| V1.0 | 增长分析与协作 | 产品、运营、市场、客服 | 可以围绕用户群、路径、渠道和预警协作 |
| V2.0 | CDP 与智能化 | 分析师、增长团队、管理层 | 可以形成标签资产、统一指标和智能诊断 |

## 3. 迭代拆分

### 迭代 1

1. SaaS 组织与项目模型
2. SDK / API 数据接收
3. 实时导入查询
4. 元事件与属性自动入库
5. 强校验与自由上报两种模式

### 迭代 2

2. 数据接入引导页
3. 上报错误样例查看
4. 测试项目 / 正式项目逻辑
5. 基础 RBAC

### 迭代 3

1. 事件分析
2. 属性分析
3. 基础概览
4. 分析结果保存与复用

### 迭代 4

1. 漏斗分析
2. 留存分析
3. 工作台首页
4. 概览共享

### 迭代 5

1. 用户细查
2. 分群

4. 分群保存与导出

### 迭代 6

1. 路径分析
2. 分布分析
3. 间隔分析
4. 虚拟事件

### 迭代 7

1. 渠道链接管理
2. 智能预警第一版
3. 公共概览
4. 更完整的数据权限

### 迭代 8

1. 标签系统

### 迭代 9

### 迭代 10

3. Webhook 扩展
4. 管理员运营配置中心

## 4. 版本升级策略

| 当前版本 | 升级条件 | 升级后新增价值 |
| --- | --- | --- |
| V0.1 -> V0.2 | 数据接入稳定、字段质量可控 | 开始产生业务分析价值 |
| V0.2 -> V1.0 | 分析模型使用率稳定 | 开始形成运营可复用资产 |
| V1.0 -> V2.0 | 分群和预警已成为常规动作 | 开始建立企业级用户资产与统一口径 |

## 5. 推荐商业化分层

| 套餐 | 建议包含 |
| --- | --- |
| Starter | V0.1 + V0.2 基础能力 |
| Growth | Starter + V1.0 全量能力 |
| Pro / Enterprise | Growth + V2.0 能力 |

## 6. 发布门槛

### V0.1 发布门槛

1. 数据接收链路稳定
2. 元数据管理可用
3. 实时调试可用
4. 项目级隔离可验证

### V0.2 发布门槛

1. 事件分析、属性分析、漏斗、留存均可用
2. 看板可创建、编辑、分享
3. 查询缓存和异步执行策略可用

### V1.0 发布门槛

1. 分群和用户细查可闭环
2. 路径、分布、间隔分析稳定
3. 预警可送达
4. 渠道链接可闭环到分析

### V2.0 发布门槛

1. 标签与画像资产可持续沉淀
2. 归因与指标平台口径一致
3. 智能诊断结果具备可解释性
