# V0.1 PRD - 数据接入与治理

## 1. 版本目标

在 V0.1 中，OI Analytics Cloud 必须先解决“数据能不能稳定、规范、可验证地进入系统”的问题。该版本上线后，客户应能完成项目创建、SDK 或 API 接入、实时调试、元数据管理和基础权限配置。

## 2. 用户价值

1. 研发能在 1 天内打通一个新项目的数据上报
2. 数据团队能看到事件和属性是否按预期入库
3. 管理员能控制数据接入模式和基础权限
4. 业务方可开始为下一版本的分析使用准备干净数据

## 3. In Scope

1. 组织 / 租户创建与登录
2. 项目创建、编辑、删除、停用
3. 测试项目与正式项目概念
4. SDK / HTTP API 接收事件数据
5. 用户属性写入与更新
6. 实时导入数据查询
7. 数据接入引导页
8. 元数据管理
9. 事件、事件属性、用户属性的显示名与类型管理
10. 自由上报模式与强校验模式
11. 接入错误日志与样例查看
12. 基础角色权限
13. 审计日志第一版

## 4. Out of Scope

1. 复杂分析模型
2. 分群 / 标签 / 概览
3. 渠道链接
4. 智能预警
5. 归因分析

## 5. 关键概念定义

| 概念 | 定义 |
| --- | --- |
| 租户 | SaaS 下的组织级隔离单元 |
| 项目 | 一个产品或业务线的数据空间 |
| 测试项目 | 用于接入调试和验证 |
| 正式项目 | 用于生产数据分析 |
| 元事件 | 标准化行为事件定义 |
| 事件属性 | 事件携带的字段 |
| 用户属性 | 用户级 profile 字段 |
| 强校验模式 | 仅允许符合预定义元数据结构的数据入库 |
| 自由上报模式 | 允许新事件 / 新属性自动入库 |

## 7. 功能需求

### 7.1 租户与项目管理

| 编号 | 需求 |
| --- | --- |
| R-V01-001 | 系统支持创建租户，并在租户下创建多个项目 |
| R-V01-002 | 项目必须包含中文名、英文名、状态、创建时间、创建人 |
| R-V01-003 | 项目支持测试环境与正式环境两个逻辑空间，接收地址可区分 |
| R-V01-004 | 项目删除需二次确认，并默认软删除 |
| R-V01-005 | 项目状态至少包含：创建中、生效中、停用中、删除中、失败 |

### 7.2 数据接收

| 编号 | 需求 |
| --- | --- |
| R-V01-010 | 系统支持 HTTP API 接收事件数据 |
| R-V01-011 | 支持 SDK 模式上报行为事件、用户属性、登录用户绑定 |
| R-V01-012 | 每条事件至少包含 event_name、distinct_id、time、properties |
| R-V01-013 | 支持 server_url / project token 形式的接入配置 |
| R-V01-014 | 支持 HTTPS，HTTP 仅用于测试或关闭 |
| R-V01-015 | 系统必须记录原始接收结果和校验结果 |

### 7.3 实时调试与接入验证

| 编号 | 需求 |
| --- | --- |
| R-V01-020 | 开发者可在实时导入查询页查看最近 N 分钟内上报的数据 |
| R-V01-021 | 列表中需展示事件名、时间、distinct_id、项目、校验结果 |
| R-V01-022 | 支持按事件名、校验状态、接收时间筛选 |
| R-V01-023 | 点击单条记录可查看完整 JSON 负载 |
| R-V01-024 | 对校验失败数据显示失败原因 |

### 7.4 元数据管理

| 编号 | 需求 |
| --- | --- |
| R-V01-030 | 元事件、事件属性、用户属性支持自动发现 |
| R-V01-031 | 支持为事件和属性设置显示名、备注、标签、启用状态 |
| R-V01-032 | 支持搜索事件名 / 显示名 |
| R-V01-033 | 支持隐藏、停用、恢复事件 |
| R-V01-034 | 支持下载当前项目的埋点方案文档 |
| R-V01-035 | 属性名称在所属表内必须唯一，类型必须一致 |

### 7.5 接入模式控制

| 编号 | 需求 |
| --- | --- |
| R-V01-040 | 项目级支持自由上报模式与强校验模式 |
| R-V01-041 | 强校验模式下，未注册事件或属性不得入库 |
| R-V01-042 | 自由上报模式下，新事件 / 新属性可自动进入元数据池 |
| R-V01-043 | 任何被拒绝的数据都必须可追踪失败原因 |

### 7.6 权限与审计

| 编号 | 需求 |
| --- | --- |
| R-V01-050 | 至少支持管理员、开发者、分析师、只读用户四类系统角色 |
| R-V01-051 | 管理员可配置成员加入项目 |
| R-V01-052 | 接入配置、元数据编辑、项目删除等操作必须写入审计日志 |

## 8. 数据对象要求

### 8.1 Event

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| tenant_id | 租户 ID |
| project_id | 项目 ID |
| event_name | 英文事件名 |
| event_display_name | 中文显示名 |
| distinct_id | 用户唯一标识 |
| event_time | 事件时间 |
| properties | JSON 属性 |
| ingest_source | sdk / api / importer |
| validation_status | success / failed |
| validation_errors | 错误明细 |
| created_at | 创建时间 |

### 8.2 Metadata

| 对象 | 关键字段 |
| --- | --- |
| meta_event | event_name, display_name, tags, enabled |
| meta_event_property | property_name, type, display_name, unit |
| meta_user_property | property_name, type, display_name, enabled |

## 9. 接口能力要求

1. 项目管理 API
2. 数据接收 API
3. 用户属性写入 API
4. 元数据查询与编辑 API
5. 实时导入查询 API
6. 接入错误明细 API
7. 角色成员管理 API
8. 审计日志 API

## 10. 权限要求

| 动作 | 管理员 | 开发者 | 分析师 | 只读 |
| --- | --- | --- | --- | --- |
| 创建项目 | 是 | 否 | 否 | 否 |
| 查看接入引导 | 是 | 是 | 否 | 否 |
| 查看实时导入 | 是 | 是 | 否 | 否 |
| 编辑元数据 | 是 | 是 | 否 | 否 |
| 查看元数据 | 是 | 是 | 是 | 是 |
| 管理成员 | 是 | 否 | 否 | 否 |

## 11. 非功能要求

1. 接收 API 可用性 >= 99.9%
2. 实时导入查询数据延迟 <= 10 秒
3. 元数据列表支持 10 万级条目分页检索
4. 所有敏感操作必须有操作人和时间戳
5. 租户与项目数据绝不串读

## 12. 验收标准

| 编号 | 验收项 |
| --- | --- |
| AC-V01-001 | 创建项目后，系统可生成独立接收地址和 token |
| AC-V01-002 | 正确上报事件后，10 秒内可在实时导入页看到记录 |
| AC-V01-003 | 强校验模式下，非法属性类型上报会被拒绝并显示原因 |
| AC-V01-004 | 元事件显示名修改后，后续分析下拉面板显示新名称 |
| AC-V01-005 | 非授权成员无法访问项目接入数据 |

## 13. 版本完成定义

V0.1 完成的标准不是“能收数据”，而是“能被客户稳定接入、定位问题、治理元数据，并为分析做好准备”。
